# Runbook 04 — 활성 PowerShell 세션에 커맨드 입력

## 목표

목록에 표시된 PowerShell 프로세스를 선택하여,
앱에서 커맨드를 입력하고 해당 세션에 실행시킨다.

---

## 구현 가능 방법

### 방법 A — stdin 리디렉션 (RedirectStandardInput)

앱이 직접 실행한 프로세스에 한정하여 사용 가능.
`Process.Start` 시 `RedirectStandardInput = true`로 설정하면,
이후 언제든 `process.StandardInput.WriteLine("명령어")`로 프롬프트에 입력 주입 가능.

| 항목 | 내용 |
|------|------|
| 대상 | 앱이 직접 실행한 프로세스만 |
| 프롬프트 활성 중 주입 | 가능 |
| 구현 난이도 | 낮음 |
| 제약 | 기존(앱 실행 전부터 떠 있던) 프로세스 불가 |

---

### 방법 B — WriteConsoleInput (Win32 API)

Windows API로 대상 콘솔의 입력 큐에 키스트로크를 직접 밀어넣는 방식.
기존 프로세스에도 적용 가능하며, 프롬프트 활성 상태에서도 동작.

| 항목 | 내용 |
|------|------|
| 대상 | 모든 콘솔 프로세스 (기존 포함) |
| 프롬프트 활성 중 주입 | 가능 |
| 구현 난이도 | 높음 |
| 제약 | 사용자 타이핑 중 입력 섞일 수 있음. P/Invoke 필요 |

---

### 방법 C — Named Pipe 원격 접속

PowerShell 5.1+ / pwsh가 프로세스당 열어두는 로컬 Named Pipe에 연결.
`System.Management.Automation.Runspaces.NamedPipeConnectionInfo`로 C#에서 접근.
인터랙티브 프롬프트에 주입하는 방식이 아니라, 동일 프로세스 내 별도 런스페이스에서 실행.

| 항목 | 내용 |
|------|------|
| 대상 | 모든 PowerShell 프로세스 (조건부) |
| 프롬프트 활성 중 주입 | 불가 (별도 세션 컨텍스트) |
| 구현 난이도 | 중간 |
| 제약 | remoting 허용 여부에 따라 연결 불가 가능. 관리자 권한 필요할 수 있음 |

---

## 비교 요약

| | 방법 A (stdin) | 방법 B (Win32) | 방법 C (Named Pipe) |
|---|---|---|---|
| 기존 프로세스 지원 | ❌ | ✅ | ✅ (조건부) |
| 프롬프트 주입 | ✅ | ✅ | ❌ |
| 구현 난이도 | 낮음 | 높음 | 중간 |
| 안정성 | 높음 | 보통 | 보통 |
