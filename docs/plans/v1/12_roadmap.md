# 12. 실행 로드맵·일정 (Execution Roadmap)

> 담당: plan_roadmap_planner · 깊이: deep · 5 단계(P0 + L0~L3) · 4 적층 레이어 + 횡단 X + 후속 마일스톤 2 / FR 43/43 전수 배치
> 본 문서는 FR을 레이어/단계로 전수 배치하고 Gate 기반 상대 일정을 제시한다.

## 0. 개요

### 0-1. 목적·범위

본 문서는 Control Tower v1의 **실행 로드맵 단일 원천**이다. `04`(FR 43/NFR 22·제약)·`05`(FN 53)·`09`(ENT 18)·`10`(RISK 10·아키텍처)를 입력으로, "무엇을·왜"를 "어떤 순서로·어떤 통과 조건에서" 쌓아 올릴지를 확정한다.

- **정의하는 것**: (a) 핵 MVP + 적층 레이어 모델(§1) / (b) 압축 PoC(Phase 0) 실험 설계와 Gate0(§2) / (c) 레이어별 산출·진입 Gate(§3) / (d) 데이터·기능 선행성과 레이어 간 의존(§4) / (e) Gate 기반 상대 일정 표기 원칙(§5).
- **정의하지 않는 것(경계)**: FR/NFR·FN·ENT·RISK는 참조 전용(재번호 금지). 스택·아키텍처·리스크 완화 상세는 10, 화면 배치는 07, 영속 스키마는 09 소관.
- **핵심 제약 반영**: C2(터미널 substrate가 0순위 — 모든 오케스트레이션이 세션 I/O 소유에 의존) → 터미널 substrate가 최하층. C1(앱-소유 세션만 제어)·C4(IPC 재사용)·C6(토큰=jsonl)·C7(로컬·인증 없음)·C8(ClickOnce)·C9(Feature×Layer)가 적층 순서의 경계 조건이다.
- **FR 전수 배치 불변식(★최우선)**: FR-001~043 전량이 ≥1 레이어/단계에 배치된다(미배치 FR = 0, §1-2·§6).

### 0-2. 로드맵 모델·ID 체계

- **적층 모델**: 압축 PoC(Walking Skeleton) → 핵 MVP → 전체 기능 → 확장. 아래에서 위로 적층하며, 각 단계는 직전 단계의 **진입 Gate** 충족 시 착수한다.
- **단계 라벨(비레지스트리 내부 라벨)**: `P0`(압축 PoC), `L0`(터미널 substrate/TRM), `L1`(세션·프로파일/SES·PRF = 핵 MVP), `L2`(IPC), `L3`(관측·자산/OBS·AST), `X`(횡단 SEC·SYS), `M-PANE`·`M-RESTORE`(후속 마일스톤). 레이어/단계 라벨은 본 문서 내부 라벨이며 레지스트리 네임스페이스가 **아니다**(발번하지 않음).
- **Gate 라벨(비레지스트리)**: `Gate0`~`Gate4`. 각 Gate는 선행 단계의 관찰 가능한 종료 조건이자 후행 단계의 진입 조건이다.
- **참조 ID**: `FR-###`(04)·`FN-{CAT}-##`(05)·`ENT-###`(09)·`RISK-###`(10)은 상위 레지스트리 값을 **인용만** 한다(재번호·재배정 0, 고아 참조 0). 본 문서는 신규 레지스트리 ID를 발번하지 않는다.

### 0-3. 표기 규칙 (Gate·상대 일정)

- **[GateN]**: 단계 전이 조건. 정량 판정 기준을 함께 명시한다(§2-3·§3).
- **상대 일정**: 캘린더 날짜·주차를 고정하지 않는다. 순서는 "Gate 충족 → 다음 단계 착수"의 상대 관계로만 표기한다.
- **우선순위**: FR의 MoSCoW(04 §0-3)를 승계 — Must는 핵 MVP(L1) 이하 경로, Should는 전체 기능(L2~L3), Could는 후속 마일스톤(M-*).
- **산출 경계**: 각 레이어의 "핵심 산출물"은 그 단계가 종료될 때 관찰 가능해야 하는 결과물이다.
- **엔진 전제**: 임베드 터미널 엔진(공식 Windows Terminal 렌더러 임베드)은 VT/ANSI 처리·GPU 셀 렌더·스크롤백·선택 복사·alt-screen/TUI 렌더를 기본 제공한다. 따라서 alt-screen(FR-005) 렌더는 L0에 포함되며, 커맨드/스킬 주입(FR-014·027)은 입력 경로로 substrate 단계부터 가용하다(렌더 완성도와 무관).

## 2. Phase 0 — 압축 PoC (Walking Skeleton)

C2(터미널 substrate 0순위)에 따라 모든 오케스트레이션이 임베드 세션 I/O 소유에 의존한다. Phase 0은 이 substrate를 **최단 경로로 종단 관통**하는 vertical-slice로, 출시 대상이 아니라 상위 레이어(L1~L3)를 착수해도 안전한지 판정하는 게이트다. RISK 상위 항목에 집중해 핵심 불확실성을 제거한다.

|---|---|:--:|---|

| RISK-004 | 주입/캡처 스레딩·주입-타이핑 공존 | 중 | 커맨드/스킬 주입 경로(FR-014·027)가 substrate부터 가용한지 확인(≤100ms·타이핑 공존) |

| 7 | 게시본이 클린 머신에서 native 동봉으로 실행·렌더된다 | ClickOnce self-contained(win-x64) 게시 → 클린 머신 실행 | 클린 머신에서 터미널 렌더 · native 로드 실패 0 | RISK-010·001 |
| 8 | beta CI 핀 버전이 재현 가능하게 restore/build된다 | `CI.Microsoft.*` 버전 정확 핀 | 핀 버전 restore/build 재현 · 라이선스 배포 조건 확인 | RISK-003 |

### 2-3. Gate0 정량 판정 (P0 → L0)

Gate0은 아래 전 항목이 관찰될 때 통과한다.

```
[ ] native 존재 — 출력 루트에 핵심 5 자산 존재 (빈 터미널 0)
[ ] 빌드 경고 0
[ ] 렌더 정확 — pwsh + claude alt-screen TUI 렌더 깨짐 0
[ ] 주입 ≤100ms · 사람 타이핑 공존 (충돌 0)
[ ] 좀비 0 — 100회 spawn/kill 후 자식 프로세스 잔존 0
[ ] airspace 배치 정상 — 별도 패널 배너 + 컨텍스트 메뉴 표시
[ ] 게시본 클린 머신 렌더 (native 동봉 실행)
```

> Gate0 미충족 항목이 있으면 L0을 착수하지 않는다(substrate 불안정 상태로 상위 적층 금지). 실패 항목은 소급 RISK(§2-1)의 완화(10 §7)로 복귀한다.

### 3-2. L1 — 세션·프로파일 (SES · PRF) — 핵 MVP

- **목적**: Must FR 중심의 **최소 출시 단위**. 프로파일로 세션을 기동·제어하고 다중 세션을 관리하는 단독 가치(AI 세션 오케스트레이션의 1차 정체성)를 제공한다.
- **배치 FR**: FR-011~023 (13, Must 10 · Should 3).
- **핵심 산출물**:
  - **PRF (프로파일 애그리거트)**: 6필드 생성·편집·검증(FN-PRF-01·02), 영속화·목록·재사용(FN-PRF-03, 원자적 temp→rename), 복제·삭제(FN-PRF-04·05), 의도별 프리셋 그룹/필터(FN-PRF-06), Session.as 정의(FN-PRF-07). 신규 영속 엔티티 ENT-005·006·007·008 도입.
  - **SES (세션 소유·제어)**: 프로파일 기반 기동(FN-SES-01) + 작업디렉토리·초기명령 적용(FN-SES-02) + claude 자동실행(FN-SES-03), 임의 세션 커맨드 주입 UI(FN-SES-04, substrate 주입 경로 재사용), 정상/강제 종료(FN-SES-05·06), 런타임 상태 추적·크래시 격리(FN-SES-07·08), 다중 세션 목록·전환(FN-SES-09), 재시작(FN-SES-10). 런타임 엔티티 ENT-004.
- **선행 조건**: Gate1(L0) 통과. 데이터 선행성 — 프로파일 영속(ENT-005~008) 확립이 세션 기동(FN-SES-01)에 선행. 원자적 저장(FR-020·NFR-011)은 RISK-009 완화를 반영.
- **[Gate2] 진입 판정 (L1 → L2) — MVP 출시 게이트**:
```
[ ] 프로파일 CRUD + 원자적 영속(재시작 후 목록 유지) 동작
[ ] 프로파일 선택 → 세션 기동(cwd·초기명령·claude 자동실행 적용)
[ ] 임의 세션에 커맨드 주입(프롬프트 활성 중·다중 세션 독립) ≤100ms
[ ] 다중 세션(≥8) 목록·전환·상태 추적 · 세션 크래시 격리(타 세션 무영향)
[ ] 세션 정상/강제 종료 · 종료 후 목록/탭 정합
```
> **L1 종료 = v1 최소 출시 가능 지점.** 이후 L2·L3는 Gate 통과 시 적층한다.

[ ] 스킬 트리거 주입 → 대상 세션 스킬 발동 후 채널 송수신

## 4. 의존성·선행 조건

### 4-1. 데이터 선행성 (ENT)

```
[R] 런타임 substrate (L0)      ENT-001·002·003·004     엔진 호스팅 시 성립
        |
[P] 프로파일 애그리거트 (L1)    ENT-005·006·007·008     세션 기동의 선행 데이터
        |  spawn (value-copy)
[R] 세션 (L1)                  ENT-004                  as·pid·state 확립
        |  as-join / as-map
[X] 채널 투영 (L2)             ENT-009·010·011          세션 as로 멤버십·송신 조인
[X]+[C] 트랜스크립트·토큰 (L3)  ENT-012·013              세션 as → jsonl 매핑 → 집계
[X] 자산 투영 (L3)             ENT-014                  경로 가드(FN-SEC-04) 선결
[P] 설정·복원·진단 (X 횡단)    ENT-015·016·017·018      전 레이어 축적

캡션: 앱이 스키마를 소유하는 **영속 데이터(프로파일 ENT-005~008)** 가 L1의 선행 데이터이며, 그 위에서만 런타임 세션(ENT-004)이 성립한다. 채널·트랜스크립트·자산은 외부 소유 **투영**이라 세션의 `as`(IPC 식별자)를 키로 논리 조인하므로, L2·L3는 L1의 `as` 확립(FN-PRF-07·FN-SES-01)에 의존한다.

             +--------- X 횡단 (SEC·SYS) : P0부터 전 레이어 관통 ---------+
             |                                                            |
   P0 ==Gate0==> L0 ==Gate1==> L1 (MVP) ==Gate2==> L2 ==Gate3==> L3 ==Gate4==> M-PANE / M-RESTORE
   |             |             |                    |             |
   substrate     터미널        세션·프로파일        채널 협업     관측·자산
   검증(RISK)    substrate     (핵 MVP·단독가치)    (as 조인)     (jsonl·트리)
                 (엔진 호스팅) (프로파일 선행)      (파일 계약)   (증분·투영)
```
캡션: 직렬 적층. 각 Gate는 후행 착수의 선결 조건이다. L1(MVP)까지는 필수 경로이며, L2·L3는 Gate 통과 시 적층, M-\*는 안정화 후 선택.

## 5. 일정 표기 원칙 (상대 일정·Gate)

- **캘린더 비사용**: 착수일·완료 주차·마감일을 고정하지 않는다. 모든 전이는 "직전 단계의 진입 Gate 충족 → 다음 단계 착수"의 **상대 순서**로만 표기한다.
- **Gate 기준**: 각 Gate(Gate0~Gate4)는 §2-3·§3의 관찰 가능한 정량 판정 체크리스트로 정의된다. 체크리스트 전 항목 충족 전에는 다음 단계를 착수하지 않는다.
- **임계경로**: Gate0(RISK-001 native 배포)이 substrate 임계경로다. 이 게이트가 열려야 L0 이상이 진행된다.
- **MVP 분기점**: Gate2(L1 종료)가 최소 출시 가능 지점이다. 이후 L2·L3·M-\*는 우선순위(Must→Should→Could)와 여력에 따라 순차 적층한다.
- **횡단 진행**: X(SEC·SYS)는 별도 단계가 아니라 각 레이어 산출에 동반된다(§3-5의 활성 시점). SEC 강제(FR-038·039)·배포(FR-041)는 P0부터, 셸 조합(FR-040)은 L0부터, 위험 가드·설정(FR-037·042)은 L1부터 동반한다.
- **역행 규칙**: 어떤 Gate라도 판정 실패 항목이 발생하면 해당 항목의 소급 RISK 완화(10 §7)로 복귀하며, 상위 레이어 적층을 보류한다.
