# 06. 행동 시나리오·Journey Map (Behavior Scenarios / Journey Map)

> 담당: plan_behavior_designer · 깊이: deep · 총 BS 22 / JM 5 (UT-002 6 · UT-003 5 · UT-004 5 · UT-005 3 · UT-001 3)
> 본 문서는 UT(운영 모드)가 FN/FR을 소비하는 행동 흐름(BS)과 대표 여정(JM)을, 03의 모드 전환 루프·인간↔AI세션↔IPC 다-액터 구조 위에서 설계한다.

## 0. 개요

### 0-1. 목적·범위

본 문서는 `03_users.md`의 사용자 유형(UT-001~005·P-001)과 `05_functions.md`의 기능(FN 53, 소급 FR 43)을 **행동의 시간 흐름**으로 엮어, 누가·언제·무엇을·어떤 순서로 하는지를 행동 시나리오(BS)와 Journey Map(JM)으로 정형화한다.

- **정의하는 것**: UT가 FN/FR을 소비하는 트리거→행동→성공/이탈 흐름(BS) / 모드 전환·다-액터 협업을 관통하는 대표 여정(JM) / 행동 패턴·엣지 케이스.
- **발번 원칙**: `BS-###`·`JM-###`만 신규 발번한다. UT/P(03)·FN(05)·FR/NFR(04)·카테고리는 **참조 전용**(재번호 금지). 화면(SC)은 07에서 확정될 **후보**로만 표기한다.
- **다-액터 반영(03 §4)**: 물리적 사용자는 1명(C7)이며, UT-002~005는 동일인의 **운영 모드 facet**이다. 진짜 다-액터는 **인간 파워유저 1 ↔ 앱-소유 AI 세션 N(비인간 액터) ↔ 세션↔세션 IPC 협업**이다. BS/JM은 이 두 층(모드 전환 + 인간↔세션↔IPC)을 모두 흐름으로 표현한다.
- **터미널 렌더 전제(04 C3·C10)**: 임베드 터미널은 공식 Windows Terminal 렌더러(EasyWindowsTerminalControl)로 색·커서·스크롤백·**alt-screen/claude TUI**를 모두 실시간 정상 렌더한다. 따라서 본 문서의 모든 claude TUI 사용 시나리오는 **화면이 정상 렌더된다는 전제** 위에서 관찰·조작·주입을 서술한다. 커맨드/스킬 주입(FN-SES-04·FN-IPC-05)의 **입력 경로는 렌더 완성도와 독립**으로 항상 가용하다(FR-002).
- **경계**: FR/NFR·카테고리=04 / FN=05 / UT·P=03 / 화면(SC)·IA=07(후속). 본 문서는 "행동·여정"만 정의하고 상위 ID를 재발번하지 않는다.

### 0-2. 시나리오 ID 체계·유형 분포

- **BS-### (행동 시나리오)**: 3자리 zero-pad. 하나의 트리거로 시작되는 단위 행동 흐름. 각 BS는 **≥1 UT를 대상**으로 하고 **≥1 FN/FR을 소비**한다(고아 시나리오 금지).
- **JM-### (Journey Map)**: 3자리 zero-pad. 복수 BS를 관통하는 국면 여정(인지→탐색→온보딩→핵심사용→반복→확장). 각 JM은 소속 BS를 인용한다.

|---|---|---:|---|
| UT-002 | 세션 함대 오케스트레이터 | 6 | BS-001~006 |
| UT-003 | 개별 세션 운영자 | 5 | BS-007~011 |

| **합계** | — | **22** | — |

### 0-3. 표기 규칙

- **행동 흐름 ASCII**: 박스 내부는 ASCII 토큰만(영문 대문자·`< >`분기·`|`경로), 한글 설명은 박스 밖 캡션에 둔다(rule_visualization_guide).
  - `[STEP]` 행동 단계 · `<BRANCH?>` 분기 · `-> ` 순방향 · `| yes/no` 분기 경로 · `[SUCCESS]`/`[EXIT]` 종점.
- **액터 약물**: `PU`=인간 파워유저 · `SESS-x`=앱-소유 AI 세션(비인간 액터) · `IPC`=파일 채널 · `APP`=Control Tower 앱.
- **감정 수치(deep)**: 매우불만 -2 / 불만 -1 / 보통 0 / 만족 +1 / 매우만족 +2.
- **관련 기능**: `FN-###(FR-###)` 형식으로 소비 기능·소급 요구를 병기. **화면 후보**: 07 SC 미발번 잠정 명칭.

## 3. 핵심 행동 시나리오 (BS)

### 3-1. UT-002 세션 함대 오케스트레이터

#### [BS-001] 프리셋으로 다중 세션 함대 기동
- **대상 UT**: UT-002 (⊆ UT-001) · **트리거**: 아침/작업 착수 시 의도별 세션 세트 필요
- **목표**: 저장된 의도별 프리셋(control / sangmin+noter / Proj_N)으로 여러 세션을 1클릭 함대 기동(NFR-014 ≤3 상호작용)
- **사전 조건**: ≥1 SessionProfile 영속화됨(BS-012 산출) · cwd 유효 · 앱 실행 중
- **행동 흐름**:
```
 [OPEN APP] -> [PICK PRESET GROUP] -> < CWD/PROFILE VALID? >
                                       | yes -> [SPAWN xN] -> [APPLY cwd+INITCMD] -> [CLAUDE AUTORUN?]
                                       |         |                                     | on  -> [CLAUDE UP]
                                       |         |                                     | off -> [PWSH ONLY]
                                       |         '-> [ATTACH TABS] -> [FLEET LIST SYNC] -> [FLEET READY]
                                       | no  -> [VALIDATION ERR] -> [FIX/EDIT PROFILE] -> [RETRY | EXIT]
```
캡션: 앱 실행→프리셋 그룹 선택→프로파일/작업디렉토리 유효성 분기. yes면 N개 세션 spawn·cwd/초기명령 적용·claude 자동실행 여부 분기 후 탭 부착·함대 목록 동기화로 준비 완료(성공). no면 검증 오류→프로파일 수정→재시도 또는 이탈.
- **성공 조건**: 프리셋의 모든 세션이 running·탭 표시·목록 동기화, claude 자동실행 세션은 claude 기동됨
- **실패/이탈**: cwd 무효/프로파일 손상 → 검증 오류 이탈 · 일부 세션 spawn 실패 → 해당 세션만 error, 나머지 정상(FN-SES-08 격리)
- **관련 기능**: FN-PRF-06(FR-022)·FN-PRF-03(FR-020)·FN-SES-01(FR-011)·FN-SES-02(FR-012)·FN-SES-03(FR-013)·FN-TRM-01(FR-001)·FN-TRM-08(FR-006)·FN-SES-09(FR-017)·FN-IPC-04(FR-026)
- **화면 후보(07)**: 프로파일 목록/프리셋 그룹 뷰 · 터미널 탭 바 · 세션 목록 패널

#### [BS-002] IPC 채널 개설·세션 멤버십 구성
- **대상 UT**: UT-002 · **트리거**: 함대 내 세션들을 협업시키기 위해 공용 채널 필요
- **목표**: 앱 Channel을 `channels/<ch>`와 1:1 매핑하고 세션(as)들을 멤버로 등록
- **사전 조건**: 세션 running(BS-001) · 각 세션 as 정의됨(FN-PRF-07) · channels 루트 경로 설정됨(FN-SYS-03)
- **행동 흐름**:
```
 [OPEN CHANNEL PANEL] -> [CREATE/PICK CHANNEL] -> < CH DIR MAPPED 1:1? >
                                                   | yes -> [ADD MEMBERS by as] -> [WATCH inbox.log]
                                                   |         '-> < STALE .watcher? >
                                                   |              | yes -> [RECOVER/ABSORB] -> [CH READY]
                                                   |              | no  -> [CH READY]
                                                   | no  -> [MAP CONFLICT] -> [RESOLVE MAPPING] -> [RETRY]
```
캡션: 채널 패널 열기→채널 생성/선택→앱 채널↔IPC 디렉토리 1:1 매핑 분기. yes면 as 기준 멤버 등록·inbox.log watch, stale watcher 감지 시 가드 정합 복구 후 채널 준비. no(매핑 충돌)면 매핑 해소 후 재시도.
- **성공 조건**: 채널 멤버에 세션 as 반영·inbox.log watch 활성·relay_url/커서 상태 조회 가능
- **실패/이탈**: 채널 1:1 매핑 미확정(후속 숙제 ④) → 가정 매핑으로 동작·경고 · stale `.watcher_<as>.pid` → 자동 흡수 or 사용자 동의(NFR-010)
- **관련 기능**: FN-IPC-07(FR-029)·FN-IPC-01(FR-024)·FN-IPC-04(FR-026)·FN-IPC-02(FR-024·NFR-010)·FN-PRF-07(FR-023)
- **화면 후보(07)**: 채널 상태 패널 · 채널 멤버 관리 패널

#### [BS-003] 트리거 프롬프트로 IPC 스킬 주입 → 협업 개시
- **대상 UT**: UT-002 · **트리거**: 채널 멤버 중 skill_ipc_control 미발동 세션 존재
- **목표**: 대상 세션 입력 파이프에 "IPC 통신 시작 …" 트리거 프롬프트를 주입해 세션이 스스로 IPC 스킬 발동, 세션↔세션 협업 개시(다-액터 구조 활성화)
- **사전 조건**: 채널 구성됨(BS-002) · 대상 세션 running · 프로파일 주입스킬[] 정의(FN-PRF-01) · 입력 파이프는 렌더와 독립으로 확보됨(FR-002)
- **행동 흐름**:
```
 [SELECT TARGET SESS] -> [BUILD TRIGGER from 주입스킬[]] -> < RISKY PATTERN? >
                                                            | yes -> [CONFIRM GATE] -> [APPROVE|CANCEL]
                                                            | no  -> [WRITE to STDIN PIPE]
                                                                      -> [SESS RUNS SKILL] -> [JOIN CHANNEL]
                                                                      -> [SESS-A <-> SESS-B via IPC] -> [COLLAB LIVE]
```
캡션: 대상 세션 선택→프로파일 주입스킬 기반 트리거 구성→위험 패턴 분기(BS-005 게이트 재사용). no면 입력 파이프로 트리거 write→세션이 IPC 스킬 실행·채널 참여→세션끼리 IPC 송수신으로 협업 개시. 주입은 화면 렌더 상태와 무관한 입력 파이프 경로이며(FR-002), 인간은 지시만 하고 실제 대화는 비인간 액터끼리 진행한다.
- **성공 조건**: 대상 세션이 채널 송수신 가능 상태로 전이·채널 대화 뷰에 세션 메시지 등장(FN-IPC-06)
- **실패/이탈**: 트리거 프롬프트가 세션 프롬프트 비활성 타이밍에 유실 → 재주입 · 세션이 스킬 미보유 인식 실패 → 로컬 ~/.claude 공유 확인·재시도
- **관련 기능**: FN-IPC-05(FR-027)·FN-SES-04(FR-014)·FN-IPC-03(FR-025)·FN-IPC-01(FR-024)·FN-IPC-06(FR-028)
- **화면 후보(07)**: 세션/채널 스킬 주입 버튼 · 채널 대화 뷰

#### [BS-004] 함대 커맨드 주입(개별/다중)
- **대상 UT**: UT-002(다중)·UT-003(개별) · **트리거**: 세션(들)에 동일/개별 지시 필요
- **목표**: 목록에서 대상 세션(들)을 골라 커맨드를 각 입력 파이프에 독립 주입(NFR-002 ≤100ms)
- **사전 조건**: 대상 세션 running · 활성/선택 세션 명시(주의 분산 방지, 03 §4-3)
- **행동 흐름**:
```
 [SELECT SESS(single|multi)] -> [TYPE COMMAND] -> < TARGET CONFIRMED? >
                                                   | yes -> < RISKY? >
                                                   |          | yes -> [BS-005 GATE]
                                                   |          | no  -> [WRITE each STDIN] -> [EXEC per SESS] -> [DONE]
                                                   | no  -> [ABORT: wrong active tab] -> [RESELECT]
```
캡션: 대상 세션(단일/다중) 선택→커맨드 입력→주입 대상 확인 분기. 대상 확정+비위험이면 각 세션 stdin에 독립 write·개별 실행(성공). 대상 미확정(엉뚱한 활성 탭 위험)이면 중단·재선택. 위험 패턴은 BS-005 게이트로.
- **성공 조건**: 선택 세션 각각에서 커맨드 실행·프롬프트 활성 중에도 반영
- **실패/이탈**: 활성 탭 오인식으로 잘못된 세션 주입 위험 → 대상 명시 UI로 차단 · 다중 주입 중 일부 세션 error → 격리(FN-SES-08)
- **관련 기능**: FN-SES-04(FR-014)·FN-SES-09(FR-017)·FN-SEC-01(FR-037)
- **화면 후보(07)**: 세션 커맨드 입력 바 · 세션 목록(다중 선택)

#### [BS-005] 위험 커맨드 자기-체크포인트 승인
- **대상 UT**: UT-002 · **트리거**: 주입(FN-SES-04/IPC-05) 커맨드가 위험 패턴(예 `rm -rf`) 매칭
- **목표**: SoD 붕괴(주입자=승인자=동일인, 03 §4-3) 상황에서 **오조작 자기-체크포인트**로 파괴적 명령 100% 확인 게이트 경유(NFR-007)
- **사전 조건**: 위험 패턴 정책 정의(후속 숙제 ⑧) · 가드 on
- **행동 흐름**:
```
 [INJECT REQUEST] -> [INTERCEPT] -> < MATCH RISKY PATTERN? >
                                     | yes -> [SHOW CONFIRM: cmd + TARGET(as/멀티)]
                                     |          -> < USER APPROVE? >
                                     |               | approve -> [SEND] -> [DONE]
                                     |               | cancel  -> [BLOCK] -> [EDIT/ABORT]
                                     | no  -> [SEND directly] -> [DONE]
```
캡션: 주입 요청 인터셉트→위험 패턴 매칭 분기. 매칭 시 커맨드+주입 대상(as/다중 여부)을 표시하는 확인 다이얼로그. 승인이면 전송, 취소면 차단·수정. 비매칭이면 즉시 전송. 다중 세션 대상 표기가 오조작 방지 핵심.
- **성공 조건**: 위험 명령이 승인 없이는 전송되지 않음(게이트 100% 경유)
- **실패/이탈**: 정책 미정으로 오탐/미탐 → 후속 숙제 ⑧에서 패턴·on/off 확정 · 사용자 습관적 승인(경보 피로) → 대상·영향 범위 명시로 완화
- **관련 기능**: FN-SEC-01(FR-037·NFR-007)·FN-SES-04(FR-014)·FN-IPC-05(FR-027)
- **화면 후보(07)**: 커맨드 확인 다이얼로그(대상 강조)

#### [BS-006] 함대 토큰 개관 후 세션 정리
- **대상 UT**: UT-002 · **트리거**: 작업 종료/토큰 소진 점검 후 함대 축소
- **목표**: 세션별 토큰을 개관해 불필요/과소진 세션을 정상 또는 강제 종료
- **사전 조건**: 세션 running·jsonl 트랜스크립트 존재(FN-OBS-01)
- **행동 흐름**:
```
 [OPEN FLEET TOKEN VIEW] -> [SCAN per-SESS TOKENS] -> < SESS TO CLOSE? >
                                                       | yes -> < RESPONSIVE? >
                                                       |          | yes -> [EXIT INJECT] -> [CLEANUP] -> [LIST SYNC]
                                                       |          | no  -> [FORCE KILL] -> [CLEANUP] -> [LIST SYNC]
                                                       | no  -> [KEEP RUNNING] -> [DONE]
```
캡션: 함대 토큰 뷰 열기→세션별 토큰 스캔→종료 대상 분기. 종료 대상이 응답하면 exit 주입 정상 종료, 무응답이면 프로세스 강제 kill, 이후 정리·목록 동기화. 대상 없으면 유지.
- **성공 조건**: 선택 세션 종료·목록/탭에서 제거 또는 exited 표기·타 세션 무영향
- **실패/이탈**: 트랜스크립트 부재로 토큰 미표시 → graceful(FN-OBS-01)·"측정 불가" 표기 · 강제 kill 후 고아 프로세스 → lifecycle 정리(FN-TRM-02)
- **관련 기능**: FN-OBS-03(FR-032)·FN-OBS-02(FR-031)·FN-SES-05(FR-015)·FN-SES-06(FR-015)·FN-SES-09(FR-017)
- **화면 후보(07)**: 토큰 대시보드 · 세션 목록(종료 액션)

### 3-2. UT-003 개별 세션 운영자

#### [BS-007] 활성 탭 직접 타이핑·claude TUI 조작
- **대상 UT**: UT-003 · **트리거**: 특정 세션을 손으로 직접 조작(마이크로 국면)
- **목표**: 활성 탭 터미널에 포커스해 직접 타이핑, claude alt-screen TUI를 관찰·조작
- **사전 조건**: 세션 running · 화면은 공식 WT 렌더러가 실시간 렌더(FR-004), claude alt-screen TUI도 엔진이 기본 제공으로 정상 렌더(FR-005·C10) · 키 입력·주입 경로는 렌더와 독립 가용(FR-002)
- **행동 흐름**:
```
 [SWITCH TAB] -> [FOCUS TERMINAL] -> [TYPE KEYS] -> < ALT-SCREEN(TUI)? >
                                                     | yes -> [ENGINE RENDERS TUI] -> [OBSERVE/OPERATE claude]
                                                     | no  -> [ENGINE RENDERS CELLS] -> [OBSERVE pwsh]
                                                             -> [SPECIAL KEYS Ctrl-C/ARROW/ENTER] -> [SESS REACTS]
```
캡션: 탭 전환→터미널 포커스→키 입력→alt-screen 여부 분기. TUI면 엔진이 alt-screen을 렌더해 claude 화면을 관찰·조작, 일반이면 엔진이 셀을 렌더해 pwsh 관찰. 어느 경우든 공식 WT 렌더러가 화면을 담당하며, 특수키는 VT 시퀀스로 변환되어 렌더와 독립인 입력 파이프로 세션에 반영(FN-TRM-12).
- **성공 조건**: 키 입력이 해당 세션 입력 파이프로 전달·특수키 올바른 시퀀스·화면(일반/alt-screen 공통) 실시간 갱신
- **실패/이탈**: 포커스가 다른 세션 탭에 있으면 오입력 → 활성 탭 명시로 방지 · 특수키 조합 미매핑 → VT 시퀀스 매핑 확인 (관찰 화면 상태와 무관하게 주입/제어는 입력 파이프로 전달됨)
- **관련 기능**: FN-TRM-12(FR-009)·FN-TRM-08(FR-006)·FN-TRM-07(FR-005)·FN-TRM-05(FR-004)·FN-TRM-04(FR-003)·FN-TRM-03(FR-002)
- **화면 후보(07)**: 터미널 뷰(포커스 입력) · 터미널 탭 바

#### [BS-008] 출력 선택·복사·붙여넣기 주입
- **대상 UT**: UT-003 · **트리거**: 세션 출력의 일부(경로·에러·코드)를 재사용
- **목표**: 드래그로 셀 범위 선택·논리라인 복사, 붙여넣기로 입력 주입
- **사전 조건**: 렌더된 출력 존재 · 클립보드 접근
- **행동 흐름**:
```
 [DRAG SELECT CELLS] -> [COPY] -> [LOGICAL-LINE REJOIN] -> [CLIPBOARD]
                                                            -> < PASTE TO INJECT? >
                                                                 | yes -> [WRITE STDIN] -> [SESS INPUT]
                                                                 | no  -> [USE ELSEWHERE] -> [DONE]
```
캡션: 셀 범위 드래그 선택→복사→래핑 반영 논리라인 재구성→클립보드 저장. 붙여넣기 주입이면 입력 파이프로 write, 아니면 외부 사용.
- **성공 조건**: 선택 범위가 래핑 반영 논리라인으로 복사·붙여넣기 주입 정상
- **실패/이탈**: 래핑 경계 오인식으로 라인 분절 → 논리라인 재구성 보정 · 스크롤 중 선택 범위 소실 → 뷰포트 고정 후 재선택
- **관련 기능**: FN-TRM-13(FR-010)·FN-TRM-10(FR-007)·FN-SES-04(FR-014)
- **화면 후보(07)**: 터미널 뷰(선택 컨텍스트 메뉴)

#### [BS-009] 스크롤백 조회·터미널 리사이즈
- **대상 UT**: UT-003 · **트리거**: 과거 출력 확인 또는 패널 크기 변경

- **관련 기능**: FN-SYS-04(FR-043)·FN-SES-10(FR-018)·FN-PRF-03(FR-020)·FN-SES-01(FR-011)
- **화면 후보(07)**: 시작 복원 프롬프트 · 프로파일 목록

#### [BS-022] 앱 종료 시 소유 세션 일괄 정리
- **대상 UT**: UT-001 · **트리거**: 앱 종료
- **목표**: 앱-소유 ConPTY 세션·프로세스를 일괄 정리해 고아 프로세스 없이 종료(범위: 앱-소유만, FN-SEC-03)

- **행동 흐름**:

```
 [APP CLOSE REQUEST] -> < OWNED SESS RUNNING? >

```

- **실패/이탈**: 무응답 세션 정리 실패 → 강제 kill 폴백 · 미저장 자산/프로파일 → 종료 전 저장 확인
- **관련 기능**: FN-TRM-02(FR-001)·FN-SES-05(FR-015)·FN-SEC-03(FR-039)

## 5. 행동 패턴 분석

시나리오·여정을 횡단해 도출한 패턴. 각 패턴에 개선 방향 1줄.

### 5-1. 사용 피크 (시간·이벤트 몰림)
- **아침 함대 셋업 피크**: 프리셋 1클릭으로 다수 세션 동시 spawn(BS-001) → spawn·claude 자동실행 동시 부하 집중. → **개선**: 순차/배치 spawn 큐잉·진행 표시로 부하 평탄화(NFR-003 동시성 목표 정합).
- **대량 출력 처리 피크**: 빌드 로그·`cat` 대용량 스트리밍(BS-007) → 화면 렌더는 공식 WT GPU 렌더러가 처리하나, 앱 측 출력 캡처·스크롤백 적재·토큰 집계 파이프라인에 부하 집중. → **개선**: 렌더는 엔진 위임으로 UI 스레드 보호, 캡처·집계는 증분 처리(NFR-005).
- **토큰 파싱 피크**: 장시간 세션의 대용량 jsonl(BS-017) → 전량 재파싱 시 지연. → **개선**: append분만 증분 파싱(NFR-005).

### 5-2. 역할 충돌 (동일 자원 복수 국면 동시 행위)
- **주의 분산 오주입**: UT-002(오케)로 지시 중 활성 탭이 다른 세션 → UT-003(운영) 문맥의 활성 탭에 오주입(03 §4-3, SoD 붕괴). → **개선**: 주입 대상 as/다중 여부 명시·확인 게이트(BS-004·005).
- **관측↔개입 전환 충돌**: UT-005 관망 중 발견한 이상을 UT-003/002로 개입할 때 대상 혼동. → **개선**: 이상 세션 강조·개입 대상 고정(BS-019).
- **자산↔런타임 충돌**: UT-004가 실행 중 세션이 참조하는 프로파일을 삭제(BS-013). → **개선**: 실행 세션 참조 유지·목록만 제거(런타임 무영향).

### 5-3. 반려 루프 (제출→검수→반려→재제출)
- **프로파일 검증 루프**: 저장→필드 검증 실패(cwd/as 유일)→수정→재저장(BS-012). → **개선**: 인라인 실시간 검증으로 왕복 최소화.
- **경로 가드 루프**: 자산 저장/CRUD→경로 탈출 차단→경로 수정→재시도(BS-014·015). → **개선**: 경계 시각화로 사전 예방(~/.claude 표기).
- **위험 커맨드 게이트 루프**: 주입→위험 매칭→취소/수정→재주입(BS-005). → **개선**: 대상·영향 명시로 승인 판단 가속(경보 피로 완화).

### 5-4. 전환 지점 (Aha / Drop-off / Conversion)
- **Aha Moment**: 프리셋 1클릭으로 함대 즉시 기동 + 트리거 주입으로 세션 협업이 자동 개시(BS-001·003). → 관제탑 장악감 확보.
- **Drop-off**: (1) 터미널 이질감 → 익숙한 Windows Terminal/VS Code 관성과 어긋나면 회귀(P-001 Churn), (2) 세팅 오버헤드 > 수동 창 관리(오버헤드 역전), (3) 관측 부정확 → 신뢰 붕괴. → **개선**: 터미널 관성 정합(NFR-015)·프리셋 즉시성·관측 정확성 확보.
- **Conversion**: 수동 다중 터미널 창 관리 → 앱-소유 관제탑으로 전면 이전(BS-020 루프 정착). → 지표: 하루 활동의 오케/운영/관측이 앱 내에서 완결.

## 7. 시나리오 요약

### 7-1. BS/JM 총계·UT별 분포

### 7-2. 전환 지점 목록

### 7-3. UT별 주요 Pain
- **UT-002 오케**: 다중 세션 오주입 위험·채널 1:1 매핑 미확정·관측 신뢰.

### 7-4. 커버리지 검증
- **모든 UT ≥1 BS**: UT-001~005 전부 ≥1 BS 대상 ✅ (UT-001은 BS-020~022, 나머지 각 3~6).
