# 11. 테스트·QA 전략 (Test & QA Strategy)

> 담당: plan_test_strategist · 깊이: deep · 테스트 레벨 4(+특수 3트랙) / AC↔TC 131건(활성 129 + 유예 2) · v1 FR AC 전수 커버
> 본 문서는 04가 freeze한 FR/NFR의 **수용 기준(AC)** 을 검증 가능한 테스트 케이스(TC)로 전개하고, ConPTY 임베드·파일 기반 IPC·WPF 렌더라는 제품 특성에 맞는 품질 게이트를 정의한다.

---

## 0. 개요

### 0-1. 목적·범위

본 문서는 `04_requirements`(FR 43·NFR 22·AC)·`05_functions`(FN 53)·`06_behaviors`(BS 22·JM 5·엣지 E1~E15)·`07_interfaces`(SC 22)를 입력으로, **"무엇이 충족되면 통과인가(AC)"를 "어떻게 검증하는가(TC)"로 변환**한다. 검증의 단위는 AC이며, 검증 대상 모든 AC는 ≥1 TC로 매핑된다(AC 전수 커버리지 = 최우선 불변식).

- **정의하는 것**: 테스트 전략·피라미드/레벨(§1~2) / ConPTY·TUI·watch의 테스트 난제 대응(§2-5) / BS 기반 시나리오·엣지 회귀(§3) / AC↔TC 전수 매핑(§4) / 품질 지표·머지 차단 게이트(§5) / 테스트 데이터·환경(§6) / CI 파이프라인 게이트 삽입(§7).
- **정의하지 않는 것(경계)**: FR/NFR·AC=04 / FN=05 / BS·엣지=06 / SC=07 / 렌더 성능 실측 목표치·아키텍처 검증·PoC=10(tech). 본 문서는 10이 확정할 성능 목표를 **게이트 임계로 연계만** 한다.
- **본 문서는 레지스트리 ID를 발번하지 않는다**: FR/NFR/FN/BS/JM/SC는 참조 전용(재번호 0·고아 참조 0). **TC는 본 문서 내부 라벨**(비레지스트리).
- **v1 검증 대상 경계**: FR-008(pane 분할·Could)·FR-043(상태 복원·Could)의 "(후속)" AC는 v1 게이트 **제외**(유예 TC로만 기록). 나머지 41 FR의 전 AC + FN 실현 NFR 9종 + 품질 목표 NFR 13종을 검증 대상으로 삼는다.

### 0-2. 표기 규칙 (AC·TC 라벨·레벨 약어)

- **AC 출처 표기**: `FR-###/AC#` 또는 `NFR-###`. 복수 AC 병기는 `·`로 구분.
- **TC 라벨**: `TC-{레벨}-##` — 레지스트리가 아닌 **문서 내부 식별자**.
- **레벨 약어**: `UNIT`(단위) · `INTG`(통합) · `E2E`(종단) · `PERF`(성능) · `SEC`(보안) · `MANUAL`(수동 탐색) · `A11Y`(접근성) · `DEFER`(유예·v1 게이트 제외).
- **유형**: 정상 / 경계 / 예외 / 부하 / 정책 / 회귀 / 유예.
- **연계 표기**: BS-###·E#(엣지)·NFR-###·C#(제약)을 근거로 병기.

### 1-1. 테스트 피라미드 + 특수 트랙

## 2. 테스트 레벨

### 2-1. 단위(Unit) — FN 단위 로직·경계·예외

최다 비중·최속 피드백. **부작용 없는 순수 로직**과 **경계/예외 분기**를 대상으로 한다. ConPTY·파일시스템·WPF 디스패처는 인터페이스로 추상화해 모킹한다.

- **엔진 경계**(TRM-04/05·NFR-018): 앱은 `ITerminalSession` 경계 계약을 검증한다 — (a) 주입/캡처/수명 호출이 계약대로 엔진에 위임되는지(스텁 엔진), (b) 엔진↔폴백 교체 시 경계 소비자 무변경. VT 파싱·셀 렌더 자체는 임베드 터미널 엔진 책임으로 앱 단위테스트 대상이 아니며 통합/시각 확인으로 커버한다.
- **jsonl 토큰 파서**(OBS-01/02·NFR-020): 정상/미지 필드/손상 라인/부재/증분(tail) 픽스처로 파싱·집계 정확성과 방어적 skip을 검증.
- **프로파일 검증**(PRF-02): 필수·cwd 유효성·as 채널 내 유일성 규칙.
- **경로 가드**(SEC-04·NFR-008): `..`/심볼릭/절대경로 탈출 페이로드 배터리로 정규화·경계 강제.
- **위험 패턴 매처**(SEC-01·NFR-007): 위험/비위험/경계 패턴 셋, 정책 on/off.
- **VT 특수키 인코딩**(TRM-12)·**논리라인 재구성**(TRM-13)·**리사이즈 열/행 산출**(TRM-09)·**to 컨벤션 파서**(IPC-03)·**채널 매핑 규칙**(IPC-07)·**세션 상태 머신**(SES-07).
- **아키텍처 정적 검사**(NFR-016/017): Shell 무비즈니스로직·Feature 간 직접참조 0·IPC 전송로직 재구현 0을 정적 규칙(어셈블리 참조 그래프)으로 검증.

### 2-2. 통합(Integration) — 프로세스·파일·모듈 경계

ConPTY 실프로세스·실파일을 사용(모킹 최소화). **Windows CI 러너 필수**(ConPTY는 `CreatePseudoConsole` 의존, C2/NFR-019).

- **ConPTY 결선**(TRM-01~03·SES-04): pwsh spawn → 입출력 파이프 바인딩 → stdin write→프롬프트 반영 → stdout→파서 도달 → 파이프 오류→error 전이. 결정론적 출력 명령(`echo`)을 주입해 왕복 검증.
- **세션 lifecycle**(SES-05~08·NFR-009): 정상 종료(exit)·강제 kill·무응답 폴백(E10)·크래시 격리(E6)·상태 이벤트 반영.
- **IPC 파일 계약**(IPC-01~03·NFR-017): 실제 `channels/<ch>/` 디렉토리에서 `send.cmd` 호출→`inbox.log` append 확인, watch 변경 감지, `.relay_url`/`.cursor_<as>`/stale `.watcher_<as>.pid`(E7) 처리. **재구현 없이 기존 스크립트/파일만 소비**함을 검증.
- **원자적 저장**(PRF-03·AST-03·NFR-011): temp→rename 커밋, write 중 강제 중단(E14) 후 원본 무손상.
- **배포 스모크**(SYS-02·NFR-021): ClickOnce 게시물 설치/실행/업데이트.

### 2-3. E2E — BS/JM 흐름 종단

07의 Flow A~F(=JM-001~005 + 라이프사이클)를 SC 전이로 종단 검증. WPF UI 자동화(UIAutomation/FlaUI류) + 실 ConPTY. 느리고 비싸므로 **핵심 행복경로 + 대표 이탈 분기**만.

- Flow A(JM-001): 복원 제안→프로파일 편집·검증→프리셋 1클릭 함대 기동→탭/목록/채널.
- Flow B(JM-002): 채널 매핑→멤버·watch→트리거 주입(위험 게이트)→세션↔세션 협업→오케 주입.
- Flow C(JM-003): 탭 선택→포커스 타이핑→선택복사→스크롤백/리사이즈→종료/재시작.
- Flow D(JM-004): 관측 관망→이상 감지→모드 전환 개입→복귀.
- Flow E(JM-005): 자산 트리→에디터 편집→경로 가드+원자 저장→CRUD→설정.
- Flow F: 시작 복원→업데이트 알림→종료 정리.

## 3. 테스트 시나리오 도출 (BS 기반: 정상·경계·예외)

### 3-1. 대표 시나리오 분해

**S1 — 프리셋 함대 기동 (BS-001 / Flow A / SC-14→07/08→11)**
- 정상: 프리셋 그룹 1클릭 → N세션 spawn → cwd/초기명령/claude 자동실행 적용 → 탭·목록 동기(≤3 상호작용, NFR-014). → TC-E2E-06·TC-INTG-11·TC-PERF-06
- 경계: 세션 다수 동시 spawn 부하(배치 큐잉·8세션, NFR-003/012). → TC-PERF-06·08
- 예외: cwd 무효/프로파일 손상 → 검증 오류 이탈(E3) · 일부 spawn 실패 → 해당 세션만 error·나머지 정상(격리, E6/NFR-009). → TC-UNIT-14·TC-SEC-10(격리)

**S2 — 트리거 주입 협업 개시 (BS-003 / Flow B / SC-19→13→17)**
- 정상: 대상 세션 선택 → 주입스킬[] 기반 트리거 구성 → 입력 파이프 주입 → 세션 스킬 발동·채널 참여·세션↔세션 IPC. → TC-INTG-37·TC-E2E-08
- 경계: 다중 대상/프롬프트-ready 타이밍. → TC-INTG-16
- 예외: 프롬프트 비활성 타이밍 유실(E11) → 재주입·ready 확인. → TC-INTG-37(예외 분기)

**S3 — 함대 커맨드 주입 + 위험 게이트 (BS-004·005 / SC-12→13)**
- 정상: 대상 명시(활성/다중) → 비위험 → 각 stdin 독립 주입·개별 실행(≤100ms, NFR-002). → TC-INTG-15·17·TC-PERF-02
- 경계: 다중 세션 대상 수·프롬프트 활성 중 주입. → TC-INTG-16·17
- 예외: 활성 탭 오인식(오주입, E2) → 대상 확정 차단 · 위험 패턴 → 게이트 100% 경유(NFR-007). → TC-SEC-03·04·TC-UNIT-28

**S4 — 세션 크래시 격리·복구 (BS-011 / Flow D / SC-06·11)**
- 정상: 정상 running 유지·상태 배지. → TC-UNIT-12
- 예외: PTY 비정상 종료(E6) → 해당 세션만 error·앱/타 세션 무영향(격리 100%, NFR-009)·진단 로그(NFR-022) → 재시작 복구. → TC-SEC-10·TC-INTG-51·55·TC-INTG-22

**S5 — 프로파일 생성·검증·영속화 (BS-012 / Flow A / SC-15)**
- 정상: 6필드 작성 → 검증 통과 → 원자적 저장 → 목록 등장. → TC-UNIT-13·14·TC-INTG-24·26
- 경계: as 채널 내 유일성·cwd 경계값. → TC-UNIT-14·17
- 예외: cwd 무효/as 중복 → 인라인 반려 루프 · write 중 중단(E14) → temp→rename 손상 0. → TC-INTG-27

**S6 — 자산 편집·경로 가드·원자 저장 (BS-014·015 / Flow E / SC-21·22)**
- 정상: 트리 브라우징 → 읽기 → 편집 → 경로 가드 통과 → 원자 저장. → TC-INTG-43·44
- 경계: 대용량 파일 로드(스트리밍). → TC-PERF-04
- 예외: `..` 경로 탈출(E8) → 정규화·차단·거부(NFR-008) · 삭제 오조작 → 확인 프롬프트. → TC-SEC-02·09·TC-E2E-10

**S7 — 백그라운드 토큰 관망 (BS-017 / Flow D / SC-20)**
- 정상: 트랜스크립트 탐지 → 증분 파싱 → 세션 누적 집계·표시(read-only). → TC-UNIT-24·25·26
- 경계: 100MB jsonl 증분(≤500ms, NFR-005)·8세션 동시 집계. → TC-PERF-03
- 예외: 트랜스크립트 부재(E5) → graceful "측정 불가" · 손상 라인/미지 필드(NFR-020) → 방어적 skip. → TC-UNIT-23·32

**S8 — 앱 종료 소유 세션 일괄 정리 (BS-022 / Flow F / SC-04)**
- 정상: 소유 세션 exit·PTY close·핸들 반환(고아 0). → TC-INTG-02
- 경계: 미저장 자산/프로파일 → 저장 유도(E15).
- 예외: 무응답 세션(E10) → 강제 kill 폴백 · 외부 프로세스는 대상 아님(SEC-03 범위 강제, E1). → TC-INTG-19·TC-SEC-01·07

## 5. 품질 지표·게이트 (NFR 연계)

### 5-1. 커버리지 목표

| 대상 | 지표 | 목표 | 근거 |
|---|---|---|---|
| 전체 | 라인 커버리지 | ≥70% | 유지보수성 기준선 |
| 핵심 도메인 | 분기 커버리지 | ≥85% | 파서 경계·jsonl 파서·프로파일 검증·경로 가드·위험 매처·세션 상태 머신·IPC 계약 |
| 보안 크리티컬 | 정의 케이스 | 100% | 경로 가드(SEC-04)·포트 정책(SEC-02)·소유 범위(SEC-03)·위험 게이트(SEC-01) |
| E2E | 핵심 흐름 | Flow A·B PASS 필수 | 아침 셋업·협업 개시(Aha 경로) |

> WPF 뷰(View)·XAML 바인딩·순수 표시 코드는 커버리지 산정에서 제외(ViewModel/Service/도메인만 집계). 사용자가 직접 코드를 작성하는 워크플로이므로 커버리지는 **핵심 도메인 우선**으로 강제한다.

### 5-3. 릴리스 기준

- 머지 차단 게이트(G1~G6·G8) 전건 GREEN.
- 성능 트랙(G7): NFR-001~005 임계(10 확정치)를 **직전 릴리스 대비 회귀 없이** 충족.
- 보안 트랙: SEC 전건 PASS + 경로 탈출 배터리·포트 스캔 0.
- 수동(G9): claude TUI 골든 스크린·터미널 UX 정합(NFR-015) 릴리스 전 승인.
- 엣지 회귀(E1~E15) 전건 PASS.
- **미확정 정책 의존 게이트**(위험 패턴 목록=후속 ⑧, 채널 1:1 매핑=후속 ④)는 확정 전까지 **보수적 기본값**(위험 게이트 always-on·가정 매핑+경고)으로 게이트 유지.

### 6-1. 테스트 데이터 전략 (픽스처·팩토리·마스킹)

| 데이터 | 형태 | 용도 |
|---|---|---|
| SessionProfile 팩토리 | 유효/필수누락/cwd무효/as중복 변형 | PRF 검증(TC-UNIT-14·17) |
| jsonl 트랜스크립트 픽스처 | 정상·미지필드·손상라인·부재·**100MB 대용량**·증분(append) | OBS 파싱·방어(TC-UNIT-22~26·32·TC-PERF-03) |
| 채널 디렉토리 픽스처 | `inbox.log`·`.relay_url`·`.cursor_<as>`·**stale `.watcher_<as>.pid`** | IPC 계약·watch·복구(TC-INTG-31~39·52) |
| VT/골든 스냅샷 | 색·커서·소거·alt-screen·claude TUI 대표 화면 | 파서·렌더 골든(TC-UNIT-01·05·TC-MANUAL-01) |
| 경로 탈출 배터리 | `..`·심볼릭·절대경로·유니코드 정규화 우회 | 경로 가드(TC-SEC-02·09) |
| 위험 패턴 셋 | `rm -rf`·대량 삭제·경계(비위험 유사) | 위험 매처(TC-UNIT-28·TC-SEC-03) |
| 스트레스 스크립트 | 10만 라인 출력·8세션 소크 | 렌더/동시성(TC-PERF-01·05·06·08) |

- **개인정보 마스킹**: jsonl 트랜스크립트는 실제 프롬프트/응답 본문을 포함할 수 있으므로 픽스처는 **합성 데이터(synthetic)** 로만 구성하고, 실 트랜스크립트를 사용해야 하는 경우 본문 필드를 마스킹/치환한다. 채널 `inbox.log` 메시지 본문도 합성.
- **결정론**: 시간 의존(watch·crash 타이밍)은 트리거 API/가상 클록으로 고정, 플래키 제거.

### 6-2. 환경 분리

| 환경 | 구성 | 실행 대상 |
|---|---|---|
| local(dev) | Windows 11·.NET 10·실 ConPTY·개발자 대화형 | 전 레벨 + 수동 탐색·골든 스크린 |
| CI | **Windows 러너**(ConPTY 필수)·헤드리스 UI 하네스(STA) | UNIT·정적·INTG·SEC·E2E 스모크 |
| staging | ClickOnce 게시 채널·별도 `~/.claude`·`channels` 루트 | 배포 스모크(TC-INTG-47·48)·릴리스 전 수동/성능 |

> ConPTY·WPF는 리눅스 헤드리스 CI에서 불가 → **Windows CI 러너 확보가 전제**(NFR-019). UI 자동화는 STA 스레드·가상 데스크톱 세션에서 구동. 성능 트랙은 **기준 하드웨어**(NFR-003 명시)에서만 판정.

## 7. CI 연계

### 7-1. 파이프라인 단계 · 게이트 삽입 위치

```
[commit/PR]
   |
   v
(1) BUILD  (warning-as-error)                         -- G1 [PR 차단]
   |
   v
(2) UNIT   (파서경계·jsonl·검증·가드·매처, 초 단위)   -- G2 커버리지 [PR 차단]
   |
   v
(3) STATIC/ARCH (Feature×Layer·IPC 재구현·파서 경계)  -- G6 [PR 차단]
   |
   v
(4) INTG   (Windows 러너: ConPTY·IPC 파일계약·lifecycle·원자성) -- G4·G5 [PR 차단]
   |
   v
(5) SECURITY (포트0·경로탈출·위험게이트·소유범위·격리) -- G3 [PR 차단]
   |
   v
(6) E2E-SMOKE (Flow A·B, UIAutomation)                -- G8 [PR 차단]
   |
   v
[merge] --------------------------------------------------------
   |
   v  (nightly / pre-release)
(7) E2E-FULL (Flow C·D·E·F) + 엣지 회귀 E1~E15
(8) PERF    (10만 라인·100MB·8세션 소크)              -- G7 [회귀 시 릴리스 차단]
(9) DEPLOY  (ClickOnce 게시 스모크 · staging)         -- NFR-021
(10) MANUAL/A11Y (claude TUI 골든 스크린·UX·배지)     -- G9 [릴리스 차단]
```

### 7-2. 게이트 배치 원칙

- **빠른 실패 우선**: BUILD→UNIT→STATIC를 PR 초반에 배치(초~분 단위), 느린 INTG/E2E는 뒤로.
- **PR 게이트(차단)**: G1·G2·G3·G4·G5·G6·G8 — 머지 전 GREEN 필수.
- **야간/릴리스 전**: PERF(G7)·E2E-FULL·엣지 회귀·DEPLOY·MANUAL(G9). 성능은 PR마다 돌리기엔 비싸므로 야간 회귀로 운영하되 임계 초과 시 릴리스 차단.
- **Windows 러너 종속**: INTG·SEC·E2E는 ConPTY/WPF 때문에 Windows 러너에서만 실행(리눅스 러너는 UNIT·STATIC만).
- **플래키 격리**: watch·crash 타이밍 테스트는 결정론 하네스로 재작성, 격리 실패 시 quarantine 후 재수정(회귀 스위트 신뢰 유지).
