# 05. 기능 정의서 (Function Specification)

> 담당: plan_function_specifier · 깊이: deep · 총 FN 53 / 8 카테고리 · Must 37 / Should 14 / Could 2 · FR 43 전수 커버(100%)
> 본 문서는 04가 freeze한 FR/NFR을 사용자 가시 기능(FN-{CAT}-##)으로 전개하고, FR 전수 커버리지·카테고리 정합·추적성을 보장한다.

---

## 0. 개요

### 0-1. 목적·범위

본 문서는 `04_requirements.md`가 발번·freeze한 **FR 43 / NFR 22**를 **구현 단위 기능(FN)** 으로 전개한다. FN은 각 FR의 "무엇을 한다"를 사용자 관점 기능으로 구체화한 것이며, **함수로 실현되는 NFR**(보안 정책·경로 가드·원자적 저장·크래시 격리·stale 복구·방어적 파싱·진단 로그 등)도 FN으로 실현한다.

- **정형화 원칙**: FR/NFR을 발명·재번호·재배정하지 않는다. frozen 카테고리(8종) 외 신규 카테고리를 만들지 않는다. FN은 frozen FR/NFR을 **입력으로만** 받는다.
- **경계(다른 문서와의 관계)**: 사용자 유형·페르소나=03(권한 주체 세분) / 화면(SC)=07 / 데이터(ENT)=09 / 터미널 엔진·아키텍처 검증=10 / 레이어·일정=12. 본 문서는 그 상위의 "어떤 기능 단위로 나뉘는가 + FR/NFR 추적성"만 정의한다.
- **미매핑 NFR 처리**: 성능(NFR-001~005)·확장성(012~013)·사용성(014~015)·유지보수성 구조(016·018)·호환성(019·021)은 "함수가 아닌 품질 목표"로, FN src에 싣지 않고 관련 FN의 목표치로 참조만 하며 **검증 책임은 10(tech)** 이 진다. FN으로 실현되는 9개 NFR만 src로 매핑한다(§2·§11 참조).

### 0-2. FN ID 체계 (plan_id_system 준수)

- **FN-{CAT}-##**: 기능 정의. `{CAT}`은 04가 freeze한 8 카테고리 코드(TRM·SES·PRF·IPC·OBS·AST·SEC·SYS)만 사용, `##`은 카테고리 내 2자리 zero-pad.
- **1:N + 전수 커버리지(불변식)**: 모든 FR은 ≥1 FN으로 덮인다(FN 없는 FR = 0). 한 FR이 복수 FN으로 전개될 수 있다.
- **카테고리 정합**: FR을 소급하는 FN의 CAT == 그 FR의 카테고리. NFR만 소급하는 FN(FN-SEC-04·FN-SYS-05)은 이 검사에서 제외하되 의미상 맞는 카테고리를 따른다.
- **고아 FN 0**: 모든 FN은 ≥1 FR 또는 NFR로 소급된다.

### 0-3. 우선순위 정의 (MoSCoW — 소급 FR/NFR 승계)

각 FN은 소급 FR(복수 시 최상위)의 등급을 승계한다. **Must**(v1 가치 필수) / **Should**(v1 출시 후 우선 보강) / **Could**(여력·후속 마일스톤).

### 0-4. 권한 약어 (표기 규칙)

단일 파워유저 앱(로컬 전용·인증 없음, 04 C7)이므로 RBAC 역할 분리가 아닌 **주체 구분**으로 표기한다. 세밀한 사용자 유형·페르소나는 03이 확정한다.

| 약어 | 주체 | 의미 |
|---|---|---|
| **U** | 파워유저(User) | 사용자가 직접 트리거하는 상호작용 기능 |
| **S** | 시스템(System) | 백엔드/자동 동작(파이프·렌더·파싱·감시·정책 강제 등, 직접 상호작용 없음) |

- **소급**: 각 FN이 실현하는 FR/NFR ID.
- **화면 후보**: 07에서 SC로 구체화될 화면 명칭(잠정). SC ID 미발번.

## 3. TRM — 터미널 임베드·렌더·탭 (FN-TRM) [L0 substrate]

임베드 터미널의 세션 소유·I/O·파싱·렌더·탭·입력을 담당한다. 모든 상위 오케스트레이션이 이 기반 위에서 동작한다(04 C2).

> **엔진 위임**: 파서·셀 렌더·스크롤백·복사·리사이즈는 **외부 터미널 엔진(10 선정) 내장 기능**이다. FN-TRM-04·05·06·10·13은 자체 구현이 아니라 엔진 기능의 노출·연결로 실현하고, FN-TRM-01(spawn)·03(파이프)은 엔진 호스팅으로, 커맨드 주입(FN-SES-04)은 엔진 경계의 입력 주입 API로 실현한다.

#### FN-TRM-01 · ConPTY 의사콘솔 생성
- 소급: FR-001 · 관련(품질): NFR-019(플랫폼 호환, 검증 10) · 우선순위: Must · 권한: U
- 설명: 사용자의 세션 생성 요청 시 앱이 `CreatePseudoConsole` 기반으로 pwsh 프로세스를 spawn하고 그 PTY를 소유한다. 외부 attach는 불가하므로 앱이 spawn한 세션만 대상이 된다.
- 핵심 동작: 세션 생성 이벤트 → PseudoConsole 핸들·pwsh 프로세스 생성 → 소유 목록 등록(FN-SES-07 상태 초기화).
- 화면 후보: 터미널 뷰 / 신규 세션 트리거(툴바·프로파일 선택)

#### FN-TRM-02 · 소유 세션 lifecycle 정리
- 소급: FR-001 · 우선순위: Must · 권한: S
- 설명: 앱 종료 시 소유 중인 모든 ConPTY 세션·프로세스를 일괄 정리하여 고아 프로세스를 남기지 않는다.
- 핵심 동작: 앱 shutdown 훅 → 소유 세션 열거 → 각 PTY close·프로세스 종료 → 핸들 반환.
- 화면 후보: (백엔드, 종료 확인 다이얼로그 연동 가능)

#### FN-TRM-03 · 입출력 파이프 바인딩
- 소급: FR-002 · 우선순위: Must · 권한: S
- 설명: 세션의 입력(주입 경로)과 출력(캡처 소스)을 바인딩한다. 입력 경로는 렌더 완성도와 무관하게 확보되어 커맨드/스킬 주입 경로(FN-SES-04·FN-IPC-05)를 조기 확보한다.
- 핵심 동작: 세션 출력이 엔진 렌더 경로로 흐름 / 입력 주입 API(엔진 경계) 노출 / 파이프 오류→세션 error 전이(FN-SES-08).
- 화면 후보: (백엔드)

#### FN-TRM-04 · VT/ANSI 렌더 위임
- 소급: FR-003 · 관련(품질): NFR-018(엔진 교체성, 검증 10) · 우선순위: Must · 권한: S
- 설명: ANSI/VT 이스케이프 시퀀스 파싱·렌더를 외부 터미널 엔진(10 선정)에 위임한다(04 C3). 엔진 접근을 엔진 경계 1곳으로 두어 교체성을 확보한다.
- 핵심 동작: 세션 출력을 엔진에 전달 → 엔진이 파싱·셀 렌더. 앱은 경계 계약만 소비하며 엔진↔폴백 교체 시 소비자 무변경.
- 화면 후보: (백엔드)

#### FN-TRM-05 · 실시간 화면 렌더
- 소급: FR-004 · 관련(품질): NFR-001(렌더 처리량)·NFR-013(확장, 검증 10) · 우선순위: Must · 권한: S
- 설명: 세션 출력을 외부 터미널 엔진이 실시간 렌더한다. 문자·전경/배경색·커서 위치가 표시된다.
- 핵심 동작: 세션 출력 → 엔진 GPU 렌더러 → 화면 갱신(색·커서). 대량 출력도 엔진이 처리(NFR-001).
- 화면 후보: 터미널 뷰

#### FN-TRM-06 · 스크롤백 조회
- 소급: FR-004 · 우선순위: Must · 권한: S
- 설명: 엔진이 보존하는 과거 출력 라인을 뷰포트로 조회한다(FN-TRM-10 연동).
- 핵심 동작: 엔진 스크롤백 버퍼 → 뷰포트 스크롤 조회 → 신규 출력 시 하단 자동 복귀.
- 화면 후보: 터미널 뷰(스크롤 영역)

#### FN-TRM-07 · alt-screen/TUI 렌더
- 소급: FR-005 · 관련(품질): NFR-013(검증 10) · 우선순위: Should · 권한: S
- 설명: alt-screen 버퍼 전환과 TUI(claude 코드 화면 등)를 엔진이 렌더한다. 엔진 기본 제공이라 주입/제어와 함께 가용하다(04 C10).
- 핵심 동작: 엔진이 alt-screen 진입/복귀를 처리 → 보조 버퍼 렌더.
- 화면 후보: 터미널 뷰(alt-screen 모드)

#### FN-TRM-08 · 세션 탭 컨테이너
- 소급: FR-006 · 우선순위: Must · 권한: U
- 설명: 여러 세션을 탭으로 담아 전환한다(단일 pane→탭 단계화, 04 C8). 탭 닫기는 세션 종료(FN-SES-05)와 연동한다.
- 핵심 동작: 세션당 탭 생성 → 탭 전환 시 해당 세션 화면/포커스 활성 → 탭 닫기→종료 연동 → 세션 목록(FN-SES-09) 동기.
- 화면 후보: 터미널 탭 바 / 터미널 영역

#### FN-TRM-09 · 리사이즈→PTY 재조정
- 소급: FR-007 · 관련(품질): NFR-015(UX 정합, 검증 10) · 우선순위: Should · 권한: U
- 설명: 터미널 뷰 크기 변경 시 `ResizePseudoConsole`로 PTY 열·행을 재계산하여 래핑을 유지한다.
- 핵심 동작: 뷰 리사이즈 이벤트 → 열/행 산출 → PTY resize 호출 → 파서/렌더 재래핑.
- 화면 후보: 터미널 뷰(리사이즈)

#### FN-TRM-10 · 뷰포트 스크롤백 조회
- 소급: FR-007 · 관련(품질): NFR-015(검증 10) · 우선순위: Should · 권한: U
- 설명: 마우스 휠·스크롤바로 스크롤백 버퍼(FN-TRM-06)의 과거 출력을 조회한다.
- 핵심 동작: 휠/스크롤 입력 → 뷰포트 오프셋 이동 → 해당 구간 렌더 → 최신 출력 시 자동 하단 복귀.
- 화면 후보: 터미널 뷰(스크롤바)

#### FN-TRM-11 · pane 분할
- 소급: FR-008 · 우선순위: Could · 권한: U
- 설명: 한 탭 내 화면을 복수 pane으로 분할해 2개 이상 세션을 동시 표시한다(후속 마일스톤, 04 C8).
- 핵심 동작: 탭 내 수평/수직 분할 → pane별 세션 바인딩 → 포커스 라우팅.
- 화면 후보: 터미널 뷰(분할 레이아웃)

#### FN-TRM-12 · 키보드 입력 포커스
- 소급: FR-009 · 관련(품질): NFR-015(검증 10) · 우선순위: Must · 권한: U
- 설명: 활성 탭 터미널에 포커스 시 키 입력을 해당 세션 입력 파이프로 전달하고, 특수키(Ctrl+C·방향키·Enter 등)를 올바른 VT 시퀀스로 변환한다.
- 핵심 동작: 키 이벤트 → VT 시퀀스 인코딩 → 입력 파이프(FN-TRM-03) write → 활성 세션 반영.
- 화면 후보: 터미널 뷰(포커스 입력)

#### FN-TRM-13 · 텍스트 선택·클립보드 복사
- 소급: FR-010 · 관련(품질): NFR-015(검증 10) · 우선순위: Should · 권한: U
- 설명: 렌더된 출력에서 드래그로 셀 범위를 선택·복사하고, 붙여넣기로 입력 주입한다. 복사 시 논리 라인(래핑 반영) 텍스트를 저장한다.
- 핵심 동작: 드래그 선택 하이라이트 → 복사 시 논리라인 재구성 → 클립보드 저장 / 붙여넣기 → 입력 파이프 주입.
- 화면 후보: 터미널 뷰(선택 컨텍스트 메뉴)

## 4. SES — 세션 생성·소유·제어 (FN-SES) [L1]

프로파일을 런타임 Session으로 기동하고, 커맨드 주입·종료·상태추적·다중 관리·재시작을 담당한다. 제어 대상은 앱-소유 세션으로 한정한다.

#### FN-SES-01 · 프로파일 기반 세션 기동
- 소급: FR-011 · 관련(품질): NFR-014(워크플로 효율, 검증 10) · 우선순위: Must · 권한: U
- 설명: SessionProfile을 입력으로 런타임 Session을 기동하고, 프로파일의 초기명령·cwd·claude자동실행·주입스킬·채널멤버십을 적용한다(오케스트레이션 중심 애그리거트, 04 C9/#9).
- 핵심 동작: 프로파일 선택 → FN-TRM-01 spawn → FN-SES-02/03 적용 → FN-IPC-04 멤버십 적용 → Session이 프로파일 참조 보유.
- 화면 후보: 프로파일 목록/기동 버튼 · 후속 숙제 ①(영속화 형태, 09)

#### FN-SES-02 · 작업디렉토리·초기명령 적용
- 소급: FR-012 · 우선순위: Must · 권한: S
- 설명: 세션 기동 시 프로파일의 작업디렉토리로 이동하고 초기명령을 순서대로 실행한다. 미지정 시 기본값(홈/기본 셸).
- 핵심 동작: 기동 직후 cd 주입 → 초기명령 배열 순차 주입 → 프롬프트 준비.
- 화면 후보: (기동 자동) / 프로파일 편집 필드

#### FN-SES-03 · claude 자동 실행
- 소급: FR-013 · 우선순위: Must · 권한: S
- 설명: 프로파일 `claude자동실행`이 켜진 경우 세션 시작 시 claude 코드 세션을 자동 기동한다. off는 순수 pwsh 유지, 실행 실패 시 세션은 유지되고 오류 관측(FN-SYS-05).
- 핵심 동작: 플래그 확인 → claude 실행 명령 주입 → 실패 시 오류 로깅, 세션 유지.
- 화면 후보: 프로파일 편집(토글) / 터미널 뷰

#### FN-SES-04 · 세션 커맨드 주입
- 소급: FR-014 · 관련(품질): NFR-002(주입 지연 ≤100ms)·NFR-014(검증 10) · 우선순위: Must · 권한: U
- 설명: 목록의 임의 세션 입력 경로에 앱에서 커맨드를 주입한다(입력 경로 재사용, 04 #10). 위험 커맨드는 FN-SEC-01 가드 경유.
- 핵심 동작: 대상 세션 선택 → 커맨드 입력 → (위험 시 확인) → 입력 파이프 write → 실행. 프롬프트 활성 중·다중 세션 독립 주입.
- 화면 후보: 세션 커맨드 입력 바 / 세션 목록 · 후속 숙제 ⑧(가드 정책)

#### FN-SES-05 · 세션 정상 종료
- 소급: FR-015 · 우선순위: Must · 권한: U
- 설명: 세션에 exit를 주입해 정상 종료하고 ConPTY·프로세스를 정리한다. 종료 후 목록/탭에서 제거 또는 exited 표기.
- 핵심 동작: 종료 요청 → exit 주입 → 프로세스 종료 대기 → 핸들 정리 → 상태 exited(FN-SES-07).
- 화면 후보: 탭 닫기 / 세션 목록 액션

#### FN-SES-06 · 세션 강제 종료
- 소급: FR-015 · 우선순위: Must · 권한: U
- 설명: 무응답 세션을 프로세스 kill로 강제 종료한다.
- 핵심 동작: 강제 종료 요청 → 프로세스 kill → PTY close → 상태 exited/error.
- 화면 후보: 세션 목록(강제 종료 액션)

#### FN-SES-07 · 세션 런타임 상태 추적
- 소급: FR-016 · 관련(품질): NFR-022(진단, FN-SYS-05 연계) · 우선순위: Must · 권한: S
- 설명: 각 Session의 profile·conpty핸들·pid·as·상태(starting/running/exited/error)를 추적한다(도메인 모델 Session 런타임).
- 핵심 동작: 상태 머신 유지 → 프로세스 종료 이벤트→상태 반영 → 목록/탭/토큰(FN-OBS-03) 갱신 소스 제공.
- 화면 후보: 세션 목록(상태 배지)

#### FN-SES-08 · 세션 크래시 격리
- 소급: FR-016 · NFR-009 · 우선순위: Must · 권한: S
- 설명: 1개 세션/PTY의 비정상 종료가 앱·타 세션에 전파되지 않도록 격리하고, 해당 세션만 error로 전이한다(격리 100% 목표).
- 핵심 동작: 파이프/프로세스 예외 캐치 → 세션 단위 격리 → error 상태 → 사용자 알림·진단 로그(FN-SYS-05), 타 세션 무영향.
- 화면 후보: 세션 목록(오류 표기) / 알림

#### FN-SES-09 · 다중 세션 목록·전환
- 소급: FR-017 · 관련(품질): NFR-003(동시성)·NFR-012(≥8 세션, 검증 10) · 우선순위: Must · 권한: U
- 설명: 앱-소유 세션 함대를 목록으로 관리하고 활성 세션을 전환한다. 목록↔탭(FN-TRM-08) 동기화.
- 핵심 동작: 소유 세션 전체 목록화 → 선택 시 활성 전환·탭 포커스 → 세션 수 증가에 선형 열화 이내(NFR-012, 10 검증).
- 화면 후보: 세션 목록 패널

#### FN-SES-10 · 세션 재시작
- 소급: FR-018 · 우선순위: Should · 권한: U
- 설명: 종료된 세션을 동일 프로파일로 재기동하고, 이전 as/채널 멤버십을 승계(옵션)한다.
- 핵심 동작: exited 세션 재시작 → 동일 프로파일로 FN-SES-01 → as/멤버십 승계 옵션 적용.
- 화면 후보: 세션 목록(재시작 액션) / FN-SYS-04 복원 연계

## 5. PRF — 세션 프로파일 (FN-PRF) [L1]

오케스트레이션 중심 애그리거트 SessionProfile의 생성·검증·영속화·복제·삭제·프리셋·as 정의를 담당한다(04 #9, 브리프 §4).

#### FN-PRF-01 · 프로파일 생성·편집
- 소급: FR-019 · 우선순위: Must · 권한: U
- 설명: SessionProfile 6필드(의도태그·초기명령·작업디렉토리·주입스킬[]·채널멤버십[]·claude자동실행)를 생성·편집한다.

- 소급: FR-019 · 우선순위: Must · 권한: U
- 설명: 필수 필드·작업디렉토리 유효성 등을 저장 전 검증한다.

- 화면 후보: 프로파일 목록 패널 · 후속 숙제 ①(영속화 형태, 09)

## 6. IPC — 채널·협업·주입 (FN-IPC) [L2]

skill_ipc_control의 GUI 프론트엔드로서 채널 파일을 watch/read·송신·멤버십·스킬 주입·대화 뷰·매핑을 제공한다. IPC 로직 재구현 금지(04 C4/NFR-017), 기존 파일 계약만 소비.

#### FN-IPC-01 · 채널 파일 watch·read
- 소급: FR-024 · NFR-017(재구현 금지) · 관련(품질): NFR-004(반영 ≤2s, 검증 10) · 우선순위: Must · 권한: S
- 설명: `channels/<ch>/`의 `inbox.log`·`.relay_url`·`.cursor_<as>`를 직접 read/watch하여 채널 상태를 앱 UI에 반영한다. IPC를 재구현하지 않고 파일 계약만 소비.
- 핵심 동작: 파일 watch 등록 → 변경 감지 → 파싱→UI 반영(FN-IPC-06). 전송/큐 로직 미구현.
- 화면 후보: 채널 상태 패널 · 후속 숙제 ⑥(web_monitor 관계)

#### FN-IPC-02 · 채널·멤버 상태 조회·복구
- 소급: FR-024 · NFR-010(stale 복구) · 우선순위: Must · 권한: S
- 설명: 채널 존재·멤버(.watcher_<as>/.read 등) 상태를 조회하고, stale `.watcher_<as>.pid`를 감지해 skill_ipc_control 가드와 정합되게 복구(자동 흡수, 사용자 동의 정책 준수)한다.
- 핵심 동작: 멤버 마커 스캔 → stale 감지 → 가드 정합 복구 제안/수행.
- 화면 후보: 채널 멤버 상태 패널

#### FN-IPC-03 · 채널 메시지 송신
- 소급: FR-025 · NFR-017(재구현 금지) · 관련(품질): NFR-014(검증 10) · 우선순위: Must · 권한: U
- 설명: 기존 `send.cmd`(a/b_watcher_monitor)로 메시지를 송신한다. relay/스크립트 재구현 없이 재사용(04 #3).
- 핵심 동작: from(as)·to·body 입력 → send.cmd 호출 → inbox.log append → 결과(id·ts) 표시. to 컨벤션(single/all/`a,b,c`) 지원.
- 화면 후보: 채널 송신 입력 바

#### FN-IPC-04 · 세션-채널 멤버십 관리
- 소급: FR-026 · 우선순위: Must · 권한: U
- 설명: 세션(as)의 채널 참여/이탈을 관리하고, 프로파일 채널멤버십[]을 런타임 세션과 연결한다.
- 핵심 동작: 세션 채널 참여 등록 → 멤버 목록에 as 반영 → 프로파일 채널멤버십[]이 기동 시 적용(FN-SES-01).
- 화면 후보: 채널 멤버 관리 패널 · 후속 숙제 ④

#### FN-IPC-05 · IPC 스킬 트리거 주입
- 소급: FR-027 · 관련(품질): NFR-002(주입 지연, 검증 10) · 우선순위: Must · 권한: U
- 설명: IPC 미보유 세션에 "IPC 통신 시작 …" 트리거 프롬프트를 대상 세션 입력 파이프로 전송하여 세션이 스스로 skill_ipc_control을 발동하게 한다(04 #10). 로컬 `~/.claude` 공유로 스킬 파일 배치 불필요.
- 핵심 동작: 대상 세션 선택 → 프로파일 주입스킬[] 기반 트리거 구성 → 입력 파이프 주입(FN-SES-04 경로 재사용) → 세션 스킬 발동.
- 화면 후보: 세션/채널 액션(스킬 주입 버튼)

#### FN-IPC-06 · 채널 대화 뷰
- 소급: FR-028 · 관련(품질): NFR-004(검증 10) · 우선순위: Should · 권한: U
- 설명: 채널 메시지를 시간순 대화로 시각화한다. from/to/body 구분·신규 메시지 실시간 추가. web_monitor 흡수 vs 병존은 후속.
- 핵심 동작: inbox.log 파싱(FN-IPC-01) → 시간순 대화 렌더 → 신규 append 실시간 추가.
- 화면 후보: 채널 대화 뷰 · 후속 숙제 ⑥

#### FN-IPC-07 · 앱 채널↔IPC channel 매핑
- 소급: FR-029 · 우선순위: Must · 권한: U
- 설명: 앱 Channel 개념을 skill_ipc_control channel과 1:1로 매핑한다(브리프 §4 가정, §7 오픈이슈). 매핑 규칙 확정으로 송수신·관측 일관성을 확보.
- 핵심 동작: 앱 Channel명 ↔ `channels/<ch>` 디렉토리 1:1 매핑 규칙 적용 → 송수신/관측이 매핑 기준 동작.
- 화면 후보: 채널 설정/매핑 뷰 · 후속 숙제 ④(1:1 매핑 최종 확정)

#### FN-OBS-01 · jsonl 트랜스크립트 탐지·매핑

- 설명: Claude Code가 남기는 세션 jsonl 트랜스크립트 파일을 탐지하고 앱 Session과 매핑한다. 파일 부재/미생성 시 graceful 처리.
- 핵심 동작: 트랜스크립트 경로 탐지 → Session↔jsonl 매핑 규칙 적용 → 미지 필드 무시·손상 라인 skip(방어적).
- 화면 후보: (백엔드) / 세션 상세

#### FN-OBS-02 · 토큰 사용량 파싱·집계
- 소급: FR-031 · NFR-020(방어적 파싱) · 관련(품질): NFR-005(증분 파싱 ≤500ms, 검증 10) · 우선순위: Must · 권한: S
- 설명: jsonl 트랜스크립트를 파싱해 input/output(및 cache) 토큰을 세션 단위로 누적 집계한다. 대용량은 신규 append분만 재파싱(증분).
- 핵심 동작: jsonl tail 증분 read → 토큰 필드 파싱 → 세션 누적 집계 → FN-OBS-03에 공급.
- 화면 후보: (백엔드) / 토큰 대시보드

#### FN-OBS-03 · 세션별 토큰 표시
- 소급: FR-032 · 우선순위: Should · 권한: U
- 설명: 집계된 토큰을 세션 목록/상세에 표시하고, 최신 트랜스크립트를 반영한다.
- 핵심 동작: 집계 결과(FN-OBS-02) 구독 → 세션 목록/상세 표시 → 갱신 시 최신 반영.
- 화면 후보: 세션 목록(토큰 열) / 토큰 대시보드

## 8. AST — 프롬프트 자산 편집 (FN-AST) [L3]

- 핵심 동작: 루트 트리 로드 → 노드 확장/선택 → 파일 선택 시 FN-AST-02. 경로 접근은 FN-SEC-04 경유.
- 화면 후보: 자산 트리 패널 · 후속 숙제 ⑤(편집 범위)

#### FN-AST-02 · 자산 파일 읽기·에디터 표시
- 소급: FR-034 · 우선순위: Must · 권한: U
- 설명: 선택한 자산 파일 내용을 에디터에 로드해 표시한다. 큰 파일도 표시 가능.
- 핵심 동작: 파일 read(경로 검증) → 에디터 버퍼 로드 → 텍스트/마크다운 표시.

#### FN-AST-03 · 자산 파일 쓰기·저장
- 소급: FR-035 · NFR-011(원자적 write) · 우선순위: Must · 권한: U
- 설명: 에디터 편집 내용을 파일로 원자적 저장한다. 성공/실패 피드백. 스키마 검증은 v1 범위 밖.
- 핵심 동작: 저장 요청 → 경로 검증(FN-SEC-04) → temp write→rename 원자 커밋 → 피드백.
- 화면 후보: 자산 에디터(저장 액션) · 후속 숙제 ⑤

#### FN-AST-04 · 자산 파일 생성·삭제·이름변경
- 소급: FR-036 · 관련: NFR-008(FN-SEC-04) · 우선순위: Should · 권한: U

- 핵심 동작: 생성/삭제/이름변경 → 경로 검증(FN-SEC-04) → 파일시스템 반영 → 트리 갱신. 삭제 확인 프롬프트.

## 9. SEC — 보안·안전 (FN-SEC) [횡단]

위험 커맨드 가드·로컬 전용 정책·앱-소유 범위 강제·자산 경로 안전을 횡단으로 강제한다. 단일 파워유저이나 다중 세션 오조작을 방지한다.

#### FN-SEC-01 · 위험 커맨드 확인 가드
- 소급: FR-037 · NFR-007(100% 확인 게이트) · 우선순위: Should · 권한: U
- 설명: 파괴적/위험 커맨드 주입(FN-SES-04·FN-IPC-05) 전 확인 단계를 둔다. 위험 패턴 목록·확인 UX·on/off는 후속 확정.
- 핵심 동작: 주입 요청 인터셉트 → 위험 패턴(예 `rm -rf`) 매칭 → 확인 프롬프트 → 승인 시에만 전송.
- 화면 후보: 커맨드 확인 다이얼로그 · 후속 숙제 ⑧(가드 정책 범위)

#### FN-SEC-02 · 로컬 전용·포트 미개방 정책
- 소급: FR-038 · NFR-006(수신 포트 0) · 우선순위: Must · 권한: S
- 설명: 앱은 외부 네트워크 수신 소켓/포트를 개방하지 않는다(멀티디바이스 relay=v2). IPC는 로컬 파일 기반만 사용.
- 핵심 동작: 수신 소켓 미생성 강제 → IPC는 파일 계약만(FN-IPC-01) → 외부 relay 비활성.
- 화면 후보: (백엔드) / 설정(정책 표기)

#### FN-SEC-03 · 앱-소유 세션 범위 강제
- 소급: FR-039 · 우선순위: Must · 권한: S
- 설명: 제어·주입·관측 대상을 앱-소유 ConPTY 세션으로 한정한다(외부 attach 불가, 04 #7). 전역 프로세스 감지로 확장하지 않는다.
- 핵심 동작: 모든 제어 API가 소유 세션 핸들만 수용 → 외부 프로세스 목록/주입 배제.
- 화면 후보: (백엔드) / 세션 목록(소유 범위)

#### FN-SEC-04 · 자산 경로 탈출 차단 가드
- 소급: NFR-008 · 우선순위: Must · 권한: S
- 설명: 자산 편집(FN-AST-01~04) 대상 경로가 `~/.claude` 밖으로 벗어나는 접근을 차단한다(경로 탈출 `..` 정규화·차단). NFR-only 실현 기능.
