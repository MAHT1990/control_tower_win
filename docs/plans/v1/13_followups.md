## 2. 미해결 3건 — 사용자 결정 필요 (우선)

MVP 이후 L2/L3 착수 전에 사용자 확정이 필요한 항목:

### ④ 앱 채널 ↔ IPC channel 매핑 규칙
- **질문**: 앱 Channel을 항상 `channels/<ch>/`와 1:1로 볼지? 한 앱 채널이 복수 물리 채널을 묶는 그룹 라우팅(`a,b,c`)을 어떻게 표현할지?
- **잠정 답**: 1:1 기본, 그룹 발신은 `to` 필드 컨벤션으로, 멤버십은 채널 단위. 다대일 묶음은 v1 밖.

### ⑥ web_monitor 흡수 vs 병존
- **질문**: 기존 `tools/web_monitor`(채널 시각화 웹앱)를 앱이 내재화할지, 외부 도구로 병존하고 앱은 파일 관측만 할지?
- **잠정 답**: v1은 앱 내 경량 대화 뷰 + web_monitor 병존. 완전 흡수는 후속.

### ⑧ 위험 커맨드 가드 정책
- **질문**: 단일 파워유저임에도 위험 명령 확인 가드를 둘지? 위험 패턴 목록·on/off·확인 UX 범위는?
- **잠정 답**: 최소 가드 Should·기본 on. 다중 세션 broadcast 주입에 우선 적용(오조작 방지가 유일 통제선 — SoD 붕괴, 03 §4-3).

|---|:--:|---|---|
